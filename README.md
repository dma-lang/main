# Capability Intelligence Agent (CIA)

Internal, trust-first consultant workbench over a four-pillar capability catalogue (851 subcapabilities)
and a canonical Jira delivery corpus. Every AI output carries a **trust envelope** (claim label · source
tier · evidence-reliability score · reasoning-chain backlink), and **no AI-derived value commits without
passing eight deterministic gates (G1–G8) + human approval**.

> Status: **scaffolding (Stage 0)**. The application is built in reviewable increments — see the approved
> plan at `/root/.claude/plans/indexed-enchanting-llama.md` and the rule sheet in [`CLAUDE.md`](./CLAUDE.md).

## Architecture (target)
- **One Cloud Run service** = FastAPI backend + built Vite/React SPA in a single `linux/amd64` container.
- **Postgres 16** (Cloud SQL) with **pgvector (dense) + tsvector (lexical)** hybrid retrieval in one engine.
- **Two-plane schema**: shared `control.*` (Alembic-migrated) + per-version `cat_<version>` data planes
  generated by the schema-mapping studio from `control.relation_def`.
- **Jobs**: Cloud Run Jobs + Scheduler + Tasks (+ DLQ). Cloud Storage, Secret Manager, Firebase Auth.
- **Models**: Gemini via **Vertex AI** (IAM, no long-lived keys), pinned by version (`config/models.yaml`).

## Developing
See **[`CLAUDE.md`](./CLAUDE.md)** for the full command list and the non-negotiable safeguards. Quick start
(local, hermetic — no cloud, no credentials, no spend):

```bash
docker compose -f docker-compose.dev.yml up   # Postgres 16 + pgvector
cd backend && uv sync && LLM_MODE=hermetic uv run pytest -q
cd ../frontend && pnpm install && pnpm test
```

## Safety posture
Secrets never live in the repo (Secret Manager only). Migrations never run on app startup. Irreversible,
costly, or prod-affecting actions (API enablement, `terraform apply`, deploys, non-dev migrations, secret
rotation, paid model calls) require explicit human approval and are blocked by a PreToolUse hook.
