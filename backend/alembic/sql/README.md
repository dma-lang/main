# Vendored baseline SQL

`control_baseline.sql` is **generated** from `docs/specs/schema.sql` (the canonical DDL) — control
plane only (the `control` schema, the 16 shared enums, the 35 `control.*` tables, their indexes, and
the `control.story_catalogue_link` view). The per-version `cat_<version>` **data plane is generated by
F4** (`bring_version_online()`), not by Alembic.

It is vendored here (rather than read from `docs/specs/`) so the migration is self-contained and
reproducible. To regenerate after a `schema.sql` change, re-run the extractor (strips comments first,
since some `schema.sql` comments contain `;`):

```
python3 - <<'PY'
import re, pathlib
clean = re.sub(r"--[^\n]*", "", pathlib.Path("docs/specs/schema.sql").read_text())
keep = lambda up: (up.startswith("CREATE TYPE")
    or (up.startswith("CREATE SCHEMA") and "CONTROL" in up)
    or re.match(r"CREATE TABLE CONTROL\.", up) or re.match(r"CREATE INDEX \w+\s+ON CONTROL\.", up)
    or re.match(r"CREATE VIEW CONTROL\.", up))
out = [c.strip() for c in clean.split(";") if c.strip() and keep(c.strip().upper())]
pathlib.Path("backend/alembic/sql/control_baseline.sql").write_text(";\n\n".join(out) + ";\n")
PY
```

The migration `0001_control_baseline` executes these statements individually (extensions and the
enum/table additions for SOW/benchmark + News are applied by their own later migrations).
